Security Algorithms

The security of a computer system is a crucial task. It is a process of ensuring the confidentiality and integrity of the OS.
A system is said to be secure if its resources are used and accessed as intended under all the circumstances, but no system can guarantee absolute security from several of various malicious threats and unauthorized access.
The security of a system can be threatened via two violations:
- Threat: A program that has the potential to cause serious damage to the system.
- Attack: An attempt to break security and make unauthorized use of an asset.
To enhance secuirty hashing algorithms are used. We have the following hashing algorithms.

SHA1

SHA-1 has historically been widely used in various operating systems and applications for tasks like checksum verification, digital signatures, and data integrity checks.Many modern operating systems have phased out the use of SHA-1 in favor of more secure hash functions such as SHA-256 or SHA-3. For example, Microsoft deprecated the use of SHA-1 in code signing certificates and SSL/TLS certificates in favor of SHA-256 due to security concerns.

SHA256

SHA-256 is extensively used in modern operating systems for password hashing, file integrity checking, code signing, and secure boot mechanisms. Its strong cryptographic properties ensure secure storage of passwords, verification of file integrity, authentication of software, and protection against boot-time attacks, thereby enhancing overall system security.

MD4

MD4 was historically used in some operating systems for tasks such as password hashing and file integrity checking. However, its vulnerabilities to collision attacks render it inadequate for modern security needs. As a result, modern operating systems have largely phased out its usage in favor of stronger cryptographic hash functions like SHA-256, which offer better protection against attacks and ensure the security of user passwords and system files.

MD5

MD4 has historically been utilized in certain operating systems for tasks like password hashing and file integrity checking. However, due to its vulnerabilities to collision attacks, modern operating systems have shifted away from its usage in favor of more secure cryptographic hash functions like SHA-256. This ensures stronger protection for user passwords and system files, enhancing overall system security.
